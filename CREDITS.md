# Créditos — skills vendored

El studio depende de skills de diseño/desarrollo de terceros, vendored dentro de
`plugins/mateo-studio/skills-lib/` para que el plugin sea autocontenido y portable.
Estas skills pertenecen a sus autores originales y se incluyen respetando sus licencias.
Para actualizarlas a su última versión upstream: `/mateo-sync`.

| Skill | Autor / origen |
|-------|----------------|
| `ui-ux-pro-max` | nextlevelbuilder — https://github.com/nextlevelbuilder/ui-ux-pro-max-skill |
| `impeccable` | pbakaus — https://github.com/pbakaus/impeccable |
| `taste-skill` | leonxlnx — https://github.com/leonxlnx/taste-skill |
| `frontend-design` | Anthropic (claude-plugins-official) |
| `gsap-*` (core, react, scrolltrigger, timeline, plugins, utils, performance, frameworks) | GreenSock (GSAP), skill oficial — MIT |
| `nextjs-app-router-patterns`, `react-state-management`, `typescript-advanced-types`, `modern-javascript-patterns`, `tailwind-design-system`, `responsive-design`, `interaction-design`, `seo-implementation`, `remotion`, `nano-banana`, `intent` | Varios autores de la comunidad de skills de Claude Code |
| `copywriting`, `pricing-strategy`, `launch-strategy` (marketing) · `startup-metrics-framework`, `micro-saas-launcher`, `inventory-demand-planning` (business) · backend/database/payments | Varios autores de la comunidad |

Si eres autor de alguna de estas skills y quieres ajustar la atribución o la inclusión,
abre un issue en el repo del plugin.

> Cada carpeta de skill conserva su propio `LICENSE`/`SKILL.md` original cuando lo trae.
> El código del studio en sí (orquestador `mateo`, los 6 agentes y el playbook) es de Oscar Matos.
